# Agent 编排引擎模板与运行视图规范

## 1. 文档目的

本文档定义 `plugins/orchestrator` 一期的模板视图与运行视图形态。

这份文档用于指导：

- 前端信息架构设计
- 模板编辑器设计
- 运行中视图设计
- 交互层级设计
- 一期开发与验收

本文档只覆盖当前已经明确的一期范围，不包含 workflow 可视化集成。

## 2. 核心结论

一期 `plugins/orchestrator` 采用以下设计原则：

- 左侧 Sidebar 分为 `Templates` 与 `Running`
- 模板视图采用 `Stage Board`
- 阶段内部采用 `Agent Board`
- 不使用复杂连线图
- 不将 workflow 作为一期主视图的一部分
- 运行中视图保留完整对话信息，但不提供底部自由输入框

## 3. 信息架构

### 3.1 Sidebar 结构

一期 Sidebar 至少分为两组：

- `Templates`
- `Running`

### 3.2 Templates 的定位

`Templates` 用于管理编排模板。

在这一组中，用户应能够：

- 查看模板列表
- 创建模板
- 打开模板
- 编辑模板

### 3.3 Running 的定位

`Running` 用于查看正在运行的编排实例。

在这一组中，用户应能够：

- 查看运行中的实例列表
- 打开某个 Run
- 查看其当前状态与执行情况
- 执行 Pause / Resume / Cancel

## 4. 一期不采用 DAG / 连线图

### 4.1 不采用连线图的原因

一期明确不采用传统流程编辑器式的 DAG 连线图。

原因如下：

- 本期重点不是低代码流程编排
- 本期重点是阶段与 Agent 的编排表达
- 连线图会把产品心智拉回 workflow / 节点编辑器
- 连线图实现成本高，但对当前理解收益有限
- 复杂连线不利于后续运行态叠加展示

### 4.2 一期的核心表达目标

一期只需要清晰表达三件事：

1. 整体分几阶段
2. 哪些阶段顺序推进
3. 哪些阶段或 Agent 可并发

这些信息不需要复杂连线即可表达。

## 5. 模板视图：Stage Board

### 5.1 定义

模板主视图采用：

`Stage Board`

它是一个从上到下的阶段板。

规则如下：

- 一行 `row` 表示一个阶段层
- 同一行中的多个 `column` 表示并发阶段
- 从上到下表示整体推进顺序

### 5.2 视觉语义

- 上下顺序 = 阶段推进顺序
- 同行并列 = 并发执行或同层任务
- 不画箭头
- 不画连线
- 复杂依赖不在图中表达

### 5.3 基本示意

```text
Row 1
[题材定位]

Row 2
[世界观设定] [人物设定]

Row 3
[长线大纲]

Row 4
[章节规划]

Row 5
[正文写作]
```

### 5.4 Stage Card 最小信息

每个 Stage Card 一期至少展示：

- 阶段名称
- 阶段类型
- 简短说明
- 状态占位
- 是否并发标识

### 5.5 点击行为

点击某个 Stage Card 后，主视图进入该阶段详情视图。

该详情视图的主内容为：

`Agent Board`

## 6. 阶段视图：Agent Board

### 6.1 定义

阶段内部不展示 llm / if / loop 等 workflow 节点图。

阶段内部采用：

`Agent Board`

规则与 `Stage Board` 保持一致：

- 一行表示一个执行层
- 同一行并列表示并发 Agent
- 从上到下表示内部推进顺序

### 6.2 基本示意

```text
Row 1
[Planner Agent]

Row 2
[Writer Agent] [Research Agent]

Row 3
[Reviewer Agent]
```

### 6.3 Agent Card 最小信息

每个 Agent Card 一期至少展示：

- Agent 名称
- Agent 角色
- 简短目标说明
- 执行方式占位
- 状态占位

### 6.4 点击行为

点击某个 Agent Card 后，不切换到新页面，而是在右侧打开一个配置面板。

## 7. Agent 配置面板

### 7.1 交互形式

Agent 的详细配置使用右侧飘出面板承载。

不建议一期使用全页跳转编辑。

### 7.2 一期最小配置项

Agent 配置面板至少应支持：

- `name`
- `role`
- `goal`
- `description`
- `prompt / policy` 引用
- 可调用的 `tools / skills`
- 是否允许派生子 Agent
- 输入来源
- 输出工件
- 完成条件
- 失败策略

### 7.3 一期关于 workflow 的处理

workflow 一期不作为主视图对象展示。

如需保留未来兼容性，可在 Agent 配置中预留：

- `executionMode`
- `workflowId`

但一期不要求在 UI 中完整展开 workflow 图。

## 8. 运行视图

### 8.1 Running 列表

`Running` 列表中每一项代表一个正在运行的编排 Run。

点击后进入该 Run 的详情视图。

### 8.2 Run 详情页定位

Run 详情页不是普通会话，也不是模板编辑器。

它是：

`编排运行详情页`

用于展示：

- 当前 Run 的状态
- 当前阶段
- 当前主 Agent
- 已启动的子 Agent
- 当前执行进展
- 完整运行对话与过程信息

### 8.3 运行中对话视图

Agent 的运行详情视图可采用类似会话的展示方式，但必须满足以下约束：

- 保留完整对话信息
- 保留工具调用信息
- 保留事件信息
- 不提供底部自由输入框

也就是说：

- 这是可回放的运行对话
- 不是自由聊天窗口

### 8.4 运行视图中的控制

运行视图至少应提供：

- `Pause`
- `Resume`
- `Cancel`

并能清晰展示：

- 当前状态
- 当前活跃 Agent
- 当前是否处于 `pause_requested`

## 9. 推荐的层级结构

一期推荐采用如下层级：

### 9.1 模板层级

1. 模板列表
2. Stage Board
3. Agent Board
4. Agent 配置面板

### 9.2 运行层级

1. Running 列表
2. Run 详情页
3. Agent 运行详情

## 10. 一期不做

为了避免过早复杂化，一期明确不做以下内容：

- DAG 连线编辑器
- 阶段间任意连线
- Agent 间复杂连线图
- workflow 可视化嵌套图
- 在运行页中提供自由对话输入
- 复杂回边、跳转、收敛图

## 11. 一期最小规则模型

为了配合 `row / column board` 形式，一期模板表达建议限制为以下规则：

### 11.1 阶段规则

- 下一行默认在上一行完成后开始
- 同一行默认可并发
- 特殊依赖不通过连线展示

### 11.2 Agent 规则

- 下一行默认在上一行完成后开始
- 同一行默认可并发
- 更复杂的依赖关系通过配置描述，不通过图展示

## 12. 运行态叠加规则

由于模板视图与运行视图都采用简化板式表达，因此运行状态也应能自然叠加在相同结构上。

一期建议至少支持以下状态表达：

- `pending`
- `running`
- `pause_requested`
- `paused`
- `completed`
- `failed`

状态应主要通过：

- 卡片高亮
- 标签
- 边框颜色
- 状态文案

来表现，而不是通过连线动画表现。

## 13. 开发要求

### 13.1 必须实现

- Sidebar：`Templates` / `Running`
- 模板页：`Stage Board`
- 阶段页：`Agent Board`
- Agent 右侧配置面板
- Running 列表
- Run 详情页
- 类会话运行记录视图
- Pause / Resume / Cancel 控件

### 13.2 必须避免

- 在一期中引入复杂连线编辑器
- 将模板页重新做成 workflow 节点图
- 将运行页做成可自由聊天窗口
- 让 workflow 成为一期主视图核心对象

## 14. 验收标准

满足以下条件后，才可认为本期 UI/交互设计达到要求。

### 14.1 模板视图验收

- 用户可在 Sidebar 中查看模板列表
- 用户可打开某个模板
- 打开后默认看到 `Stage Board`
- 点击某个阶段后可看到 `Agent Board`
- 点击某个 Agent 后右侧出现配置面板

### 14.2 运行视图验收

- 用户可在 Sidebar 中查看运行中的 Run 列表
- 点击某个 Run 后进入详情页
- 详情页以类会话形式展示完整过程信息
- 详情页无自由输入框
- 详情页可执行 Pause / Resume / Cancel

### 14.3 结构一致性验收

- 模板页与运行页不出现两套完全无关的视觉语言
- `Stage Board` 与 `Agent Board` 均采用 row / column 表达
- 不依赖连线来表达核心结构

## 15. 结论

一期 `plugins/orchestrator` 的模板与运行视图正式定义为：

- 模板页采用 `Stage Board`
- 阶段页采用 `Agent Board`
- 运行页采用类会话视图与运行控制结合
- 不引入复杂连线图
- 不将 workflow 纳入一期主视图

该设计应作为后续开发与验收的统一标准。
