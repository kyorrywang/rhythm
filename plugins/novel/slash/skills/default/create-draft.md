# Create Draft

你正在执行 `novel:create-draft`。**你必须调用子 agent 来生成章节正文，主 agent 禁止直接写正文。**

## 核心目标

生成一个或多个可继续迭代的章节草稿，要求：

- 连续性正确
- 节奏清晰
- 局部剧情有推进
- 章节结尾有继续阅读动力

## 输入来源

你收到的上下文通常包括：

- 项目配置（`<project_root>/project.yaml`）
- 当前小剧情 brief
- 设定集与大纲
- 对话与状态归档摘要
- 最近章节
- 本次用户追加要求

## 吸收已有上下文

在整理子 agent 任务之前，你必须先检查以下内容，确保不遗漏约束：

1. **项目配置** — 读取 `<project_root>/project.yaml`，重点关注：
   - `generation.chapter_mode`（串行 / 并行策略）
   - `discussion.arc_span_chapters`（每个 arc 覆盖的章节数）
   - `archive.*` 开关（是否追踪角色/物品/技能/伏笔）
2. **设定集** — 读取 bible，确认世界观硬规则和角色核心特征，作为不可违反的约束
3. **大纲** — 读取 master-outline，确认当前 arc 在整体结构中的位置和走向
4. **归档状态** — 读取 archive 中的 state-summary，确认：
   - 最近章节结尾的角色状态（谁在哪里、什么时间、什么情绪/处境）
   - 待回收的伏笔列表
   - 正在推进的伏笔状态
5. **最近章节** — 读取 chapters/ 下最近的 2-3 章，确认叙事衔接点和语气基调
6. **用户追加要求** — 逐条标记，确保全部在子 agent 任务中体现

如果发现 brief 与已有设定/归档存在矛盾，优先遵循设定集和归档，并在子 agent 任务说明中标注冲突点。

## 工作要求

### 主 agent 职责

- **必须调用子 agent 来生成章节正文**，主 agent 只负责任务整理和上下文传递，绝不直接写正文。
- 先整理一个非常详细的子 agent 任务说明，再调用子 agent。

### 子 agent 任务说明

发给子 agent 的说明中必须明确包含：

- 当前命令名称（`novel:create-draft`）
- 用户这次的追加要求
- `<project_root>/project.yaml` 的关键配置（特别是 `generation.chapter_mode` 和 `discussion.arc_span_chapters`）
- 当前剧情 brief、archive、设定集、大纲、最近章节
- **连续性状态**：最近章节结尾的角色位置、时间、情绪/处境
- **伏笔状态**：待回收的伏笔、正在推进的伏笔
- **用户禁区/红线**：brief 或对话中用户明确表示不想要的内容方向
- 目标输出文件路径
- 是单章还是多章，以及依据（见下方"多章判断"）
- 如果是多章，串行或并行策略（见下方"串行/并行策略"）

### 多章判断

是否写多章，应基于以下依据综合判断：

- `discussion.arc_span_chapters` 配置值（默认 3-5 章）
- 当前 arc-brief 覆盖的剧情范围
- 最近章节的叙事节奏（是否已到自然断点）

如果只适合写单章，就写单章；如果 arc-brief 和节奏支持，也可以写多章。不要凭感觉决定。

### 串行/并行策略

- 遵循 `generation.chapter_mode` 配置（默认 `serial`）
- `serial`：子 agent 按章节顺序逐章生成，后一章可看到前一章的输出
- `parallel`：子 agent 可同时生成多章，但需要主 agent 在任务说明中提供足够的前文摘要
- 如果配置为 `serial` 但上下文已明确适合并行（如多章之间高度独立），可以在任务说明中注明并行理由

### 写作约束

- 明确承接已有设定和最近章节
- 不要让角色行为失真
- 不要让剧情推进失去因果
- 不要用叙述性概括跳过关键场景——应该让角色在行动中展现变化，而不是用旁白解释变化
- 必须关注连续性与伏笔状态

## 章节文件命名规范

章节文件统一写入 `<project_root>/chapters/` 目录，命名规则：

- 格式：`ch<序号>.md`，序号为两位数字，零填充
- 示例：`ch01.md`、`ch02.md`、`ch10.md`
- 按文件名字典序排列即为章节顺序
- 如果目录中已有章节，新章节序号接续最大序号

## 写后自查

子 agent 写完章节草稿后，主 agent 应检查以下要点：

1. **连续性**：新章节开头是否自然承接上一章结尾（角色状态、时间、地点）
2. **设定一致性**：是否有违反 bible 中硬规则的内容
3. **伏笔状态**：是否误回收了尚未铺垫的伏笔，或遗漏了应推进的伏笔
4. **用户追加要求**：逐条确认是否全部落实
5. **叙事节奏**：章节是否有推进，还是原地踏步

如发现问题，主 agent 应再次调用子 agent 进行修正，而不是自行修改。

## 禁止事项

- 不要由主 agent 直接产出最终章节文件正文
- 不要只给梗概而不创建子 agent 任务
- 不要脱离 brief 重开新故事
- 不要忽视最近章节和归档状态
