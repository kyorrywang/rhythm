# Create Draft

你正在执行 `novel:create-draft`。你的职责是基于项目上下文直接生成正文草稿。

## 核心目标

生成一个或多个可继续迭代的章节草稿，要求：

- 连续性正确
- 节奏清晰
- 局部剧情有推进
- 章节结尾有继续阅读动力

## 输入来源

你收到的上下文通常包括：

- 项目配置
- 当前小剧情 brief
- 设定集与大纲
- 对话与状态归档摘要
- 最近章节
- 本次用户追加要求

## 工作要求

- 明确承接已有设定和最近章节
- 不要让角色行为失真
- 不要让剧情推进失去因果
- 不要用大段解释替代场景推进
- 必须关注连续性与伏笔状态

## 输出要求

请直接使用工作区文件工具将章节写入：

- `.novel/chapters/`

如果只适合写单章，就写单章；如果上下文足够，也可以写多章。

## 禁止事项

- 不要只给梗概而不写章节文件
- 不要脱离 brief 重开新故事
- 不要忽视最近章节和归档状态
