# Archive

你正在执行 `novel:archive`。你的职责是把最近一次创作轮次的成果提炼成可复用的长期上下文，包括对话归档和状态归档两部分。

## 核心目标

归档最近一次创作轮次中真正会影响后续续写的内容，分为两个维度：

### 对话归档

- 用户新增要求
- 用户否决的方向
- 已做出的关键决定
- 已创建或更新的文件
- 仍未解决的问题
- 下一次继续创作前最该知道什么

### 状态归档

提炼并更新这些关键状态对象：

- 角色状态与关系
- 关键物品的归属与变化
- 能力 / 体系 / 境界 / 进展
- 世界新事实
- 势力关系变化
- 已埋伏笔
- 已回收伏笔
- 待追踪伏笔
- 最近出现的连续性风险

## 输出要求

请直接使用工作区文件工具更新这些归档文件中你认为必要的部分：

- `<project_root>/archive/sessions/latest-session.md`：最近一次会话摘要
- `<project_root>/archive/summaries/conversation-summary.md`：长期可续写的对话摘要
- `<project_root>/archive/summaries/state-summary.md`：人物、物品、能力、伏笔等连续性状态摘要

## 执行方式

- 如需要借助子 agent 提升稳定性，可以先整理详细任务说明，再调用子 agent。
- 如果你调用子 agent，说明中必须包含：
  - 当前命令名称
  - 用户本次补充要求
  - `<project_root>/project.yaml`
  - 最近相关对话与已有归档文件
  - 当前设定、章节、归档摘要与相关文件路径
  - 目标输出文件路径
- 不要遗漏用户新增要求、否决方向、已更新文件和未解决问题。
- 归档状态时，要保证输出能被后续续写直接复用。

## 连续性检查

归档状态时，顺手检查并指出：

- 人物状态是否矛盾
- 物品归属是否冲突
- 能力推进是否异常
- 世界规则是否被破坏
- 伏笔是否被遗忘或误回收

## 风格要求

- 面向"后续能继续工作"
- 不是流水账
- 不保留无用寒暄
- 不遗漏硬约束和禁止项
- 状态归档要结构化、明确，不要写成泛泛总结
